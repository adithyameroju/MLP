# User Mental Model

## Key Insights

- Users think in "net impact" (premium + CD), not actions
- Endorsements are bundled (add + delete + modify)
- Validation before submission is critical
- Financial visibility = trust
- Users need explanation, not just data

---

## Behavioral Patterns

- Frequent small updates (<5 employees)
- Heavy reliance on Excel + manual validation
- Batch processing mindset (fortnightly cycles)
- Reconciliation-first workflows (especially brokers)

---

## Trust Signals Needed

- Timestamps
- Audit logs
- Clear success/failure states
- Financial previews before action
