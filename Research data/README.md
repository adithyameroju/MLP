# Research data (employer portal MLP)

User research and product framing for the **employer portal** MLP. **AI assistants and contributors should read these files** when planning or building portal UX, workflows, copy, or prioritization—not only the implementation in `src/`.

## Files

| File | Purpose |
|------|---------|
| `principles.md` | Design principles: clarity, transparency, reliability, efficiency |
| `mental-model.md` | How users think (net impact, bundled endorsements, trust signals) |
| `personas.md` | HR Ops, HR leader, brokers |
| `problems.md` | Prioritized problems (P0–P2) |
| `features.md` | Feature layers and P0 scope |
| `flows-system-pattern.md` | Standard action pattern (validate → preview → confirm → outcome) |
| `flows-cd-balance.md` | Ideal CD balance journey |
| `system-gaps.md` | Cross-cutting gaps |
| `success-metrics.md` | How to measure success |

## Project rule

Cursor loads `.cursor/rules/employer-portal-research.mdc` so sessions in this repo **prioritize** this folder alongside the codebase.

## Adding research

Add new `.md` files here and, if they change core principles, briefly reference them in this README table.
